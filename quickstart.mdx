---
title: "Quickstart"
description: "Get the Eventos API running locally in minutes"
---

<Note>
  The API enforces CORS and only accepts browser requests originating from `https://eventosutmach.000webhostapp.com`. Direct `curl` and server-to-server calls are unaffected by this restriction.
</Note>

<Steps>
  <Step title="Prerequisites">
    Make sure you have the following installed before continuing:

    - **Node.js 14.x** — the API targets the `14.x` engine as declared in `package.json`
    - **PostgreSQL** — a running instance accessible via a connection string
    - **git** — to clone the repository
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url>
    cd apinode
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```
    </CodeGroup>

    This installs `express`, `dotenv`, `pg`, and `cors` as listed in `package.json`.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash .env
    DATABASE_URL=postgres://user:password@host:5432/dbname
    PORT=3000
    ```

    | Variable | Required | Default | Description |
    |---|---|---|---|
    | `DATABASE_URL` | Yes | — | PostgreSQL connection string. SSL is enabled automatically. |
    | `PORT` | No | `3000` | Port the HTTP server listens on. |
  </Step>

  <Step title="Create the Prueba table">
    Connect to your PostgreSQL database and run the following DDL to create the table the API queries:

    ```sql
    CREATE TABLE IF NOT EXISTS Prueba (
      id SERIAL PRIMARY KEY,
      -- add your columns here
    );
    ```

    The endpoint executes `SELECT * FROM Prueba`, so the table must exist before the server receives requests.
  </Step>

  <Step title="Start the server">
    ```bash
    npm start
    ```

    The script runs `node index.js`. You should see the server listening on the configured `PORT` (default `3000`).
  </Step>

  <Step title="Test the endpoint">
    Send a request to `GET /lista`:

    ```bash
    curl http://localhost:3000/lista
    ```

    A successful response returns HTTP `200` with a JSON array of all rows from the `Prueba` table:

    ```json
    [
      {
        "id": 1
        // ...your column values
      }
    ]
    ```

    If the database cannot be reached, the API returns HTTP `500`:

    ```json
    { "error": "Error en el servidor" }
    ```
  </Step>
</Steps>
